When do we know when we should be lazy and look for a library, and when should we manually create new code?
The question I ask is, "Has anyone else ever had to do a task like this?" If yes, there may be a library. How much time should I spend investigating the existence and use of the library depends on the complexity of the task and how quickly I think I could come up with a manual solution.

Where do you (Ben) go to find the libraries you are showing in class?
I don't have a specific place I look (other than searching Google). Most of the libraries I show in class are because I faced a specific challenge as a data scientist, then found the library using the logic above ("Has anyone else done something similar?"). There is a dependence on knowing the relevant jargon in order to search the Internet effectively. Having the vocabulary for the topic you're interested in is vital.

What are the risks of automation?
When the system you've automated does not conform to the assumptions you've made, then your automation will fail. Keeping this in mind, design the failure mode of your automation to be loud (not silent) and safe.

What role does automation play in Artificial Intelligence (AI)?
The phrase "Artificial Intelligence" is loosely defined, so I don't have an answer. For machine learning, the typical sequence of steps is to get data, clean data, extract features, train the model or infer using the model, then present a result. In the "get data" and "clean data," automation can be applied.

When to switch languages for automation?
Depends on the task being performed and the scope of the language being used. For example, compare https://www.google.com/search?q=automate+mouse+movement+python and https://www.google.com/search?&q=automate+mouse+movement+mac -- there are tools for both approaches.

How to automate sending encrypted emails?
In class I showed how to enable TLS for SMTP. Using Google's API also requires secure authentication. These result in communication between you (the client) and the server being secure. However, once you send your email, it is plain text.
To send encrypted emails, you would separately encrypt the message body prior to sending the content to the server. See GPG and these search results.

How do you decide which libraries in Pandas are trustworthy?
I don't. I run Jupyter in a Docker container so that packages I import don't mess with my computer. When I shut down the Docker container, anything not part of the image is lost. That's bad if I was storing data, good if there was malicious code.
In general, free software does not imply trustworthy software. Opensource software (where you can inspect the code) is also not guaranteed to be either bug free or have perfect security.

Why does Selenium software often not work?
Because it is dependent on a web browser to behave in certain ways. When those assumptions are broken, Selenium doesn't function as expected. The more external dependencies the code has, the more likely it is to break.
