One of the questions last week was about integrating Python with other languages. There are a couple of ways in which this can be interpreted.

The most common way I use Python with other languages is as a step in a workflow. I often start my analysis using Linux's bash (grep, sed, awk, cut, find) or, on Windows, Powershell. I use bash to extract the information I want, then I use Python to do follow-on processing that is more complex or larger scale.

It is common to call one language from another. This is done for a couple reasons: to avoid re-implementing an existing solution, or because another language is more effective (eg faster) for the task. For example, Python and C can be integrated; see Extending Python with C or C++. Similarly, we can call R from Python. For an extensive listing of combinations, see https://wiki.python.org/moin/IntegratingPythonWithOtherLanguages

Another sense of using Python with other languages is meta-programming. In this situation, Python can be used to produce programs. A Python script can generate other Python scripts, or a Python script can produce a program written in a different language, eg Fortran. I used metaprogramming when I worked with Fortran programs to generate Fortran source code from a Python script.

We encountered metaprogramming when we discussed automation in week 5. In that lecture we reviewed how to use Python to generate HTML from templates.
